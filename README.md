# 🐋 Black Whale Academy — نظام الأنشطة والحضور

تطبيق خفيف لإدارة أنشطة أكاديمية رياضية (سباحة، كرة قدم، كاراتيه...) وتفقد حضور الأطفال، مبني بـ Next.js وقاعدة بيانات Postgres (Neon).

## المكونات

- **/register** — رابط تسجيل يُرسل للأهل (بدون باسورد، وبدون أي رابط رجوع لباقي الموقع)، يعبّوا اسم الطفل + بيانات تواصل + صورة، ويختاروا نشاط واحد أو أكتر مع باقة الحصص المناسبة، وبيتولّد للطفل كود QR خاص فيه قابل للتحميل كصورة.
- **/admin** — محمي بباسورد مشترك للموظفين. إدارة الأنشطة (الاسم، المدرب، المواعيد، وباقات الحصص/الأسعار)، وإدارة اشتراكات كل طفل (إضافة/تجديد/إلغاء بكل نشاط)، وطباعة/تحميل بطاقة QR لكل طفل.
- **/attendance** — شاشة المشرف اليومية: يختار النشاط الجاري حالياً، ثم يمسح QR من كاميرا الموبايل لتسجيل الحضور بهالنشاط تحديداً، مع إمكانية تعديل يدوي.
- **/history** — استعراض سجل أي يوم سابق، بشكل عام أو حسب نشاط معيّن.

## خطوات التشغيل

### ١. قاعدة البيانات (Neon)

1. افتح مشروع Postgres جديد على [neon.tech](https://neon.tech).
2. افتح **SQL Editor** وشغّل محتوى ملف `schema.sql` الموجود بهذا المستودع (مرة وحدة فقط).
3. انسخ الـ **Connection String** (رابط الاتصال).

### ٢. النشر على Vercel

1. افتح [vercel.com](https://vercel.com) وسجل دخول بحساب GitHub.
2. اعمل **Import Project** واختر هذا المستودع.
3. بقسم **Environment Variables** ضيف:
   ```
   DATABASE_URL = <رابط الاتصال يلي نسخته من Neon>
   ADMIN_PASSWORD = <باسورد الإدارة يلي رح تشاركه مع الموظفين بس>
   ```
4. اضغط **Deploy**. بعد ما يخلص، رح تاخد رابط live جاهز.

### ٣. الاستخدام

- شارك `<رابطك>/register` مع الأهل — ما بيحتاج باسورد.
- شارك `<رابطك>/attendance` مباشرة مع المدربين — ما بيحتاج باسورد، وما فيه أي وصول لمعلومات الإدارة أو الأسعار.
- ادخل `<رابطك>/admin` بالباسورد يلي حطيته بـ`ADMIN_PASSWORD` عشان تضيف الأنشطة وباقات الحصص، وتتابع اشتراكات كل طفل، وتطبع/تحمّل بطاقات QR.

## التطوير محلياً

```bash
npm install
cp .env.example .env.local   # وحط رابط قاعدة بياناتك وباسورد الإدارة
npm run dev
```
